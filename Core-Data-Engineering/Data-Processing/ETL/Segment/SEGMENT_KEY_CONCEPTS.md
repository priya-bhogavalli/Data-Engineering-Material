# Segment - Key Concepts

## Overview
Segment is a customer data platform (CDP) that collects, cleans, and controls customer data, then sends it to various marketing, analytics, and data warehousing tools.

## Core Architecture
- **Single API**: One API to collect all customer data
- **Sources**: Collect data from websites, mobile apps, servers
- **Destinations**: Send data to 300+ tools and warehouses
- **Transformations**: Clean and enrich data in real-time
- **Personas**: Audience building and activation
- **Protocols**: Data governance and quality controls

## Data Collection
- **Analytics.js**: JavaScript library for web tracking
- **Mobile SDKs**: iOS and Android SDKs
- **Server libraries**: Node.js, Python, Ruby, Java, PHP
- **Cloud sources**: Import data from SaaS tools
- **Pixel tracking**: Email and ad pixel tracking
- **Webhooks**: Real-time event ingestion

## Event Types
- **Track**: User actions and behaviors
- **Page**: Page views and screen views  
- **Identify**: User identification and traits
- **Group**: Associate users with groups/accounts
- **Alias**: Link user identities across devices
- **Screen**: Mobile app screen tracking

## Destinations
- **Analytics**: Google Analytics, Mixpanel, Amplitude
- **Marketing**: Facebook Ads, Google Ads, Mailchimp
- **Data warehouses**: Snowflake, Redshift, BigQuery
- **CRM**: Salesforce, HubSpot, Intercom
- **A/B testing**: Optimizely, VWO
- **Customer support**: Zendesk, Freshdesk

## Key Features
- **Real-time processing**: Sub-second data delivery
- **Schema validation**: Ensure data quality and consistency
- **Identity resolution**: Unify user identities across touchpoints
- **Audience building**: Create and sync audiences
- **Privacy controls**: GDPR and CCPA compliance
- **Replay**: Historical data replay to new destinations
