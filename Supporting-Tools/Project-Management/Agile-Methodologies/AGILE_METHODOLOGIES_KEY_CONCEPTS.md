# Agile Methodologies - Key Concepts

## Overview
Agile methodologies are iterative approaches to software development and project management that emphasize flexibility, collaboration, and rapid delivery of working solutions.

## Agile Manifesto

### Core Values
- **Individuals and interactions** over processes and tools
- **Working software** over comprehensive documentation
- **Customer collaboration** over contract negotiation
- **Responding to change** over following a plan

### Twelve Principles
- **Customer satisfaction**: Early and continuous delivery
- **Welcome change**: Embrace changing requirements
- **Frequent delivery**: Working software in short timeframes
- **Collaboration**: Business and developers work together
- **Motivated individuals**: Trust and support team members

## Scrum Framework

### Roles
- **Product Owner**: Defines requirements and priorities
- **Scrum Master**: Facilitates process and removes impediments
- **Development Team**: Cross-functional team building product
- **Stakeholders**: Users and business representatives

### Events
- **Sprint**: Time-boxed iteration (1-4 weeks)
- **Sprint Planning**: Plan work for upcoming sprint
- **Daily Scrum**: Daily synchronization meeting
- **Sprint Review**: Demonstrate completed work
- **Sprint Retrospective**: Reflect and improve process

### Artifacts
- **Product Backlog**: Prioritized list of features
- **Sprint Backlog**: Work selected for current sprint
- **Increment**: Potentially shippable product increment
- **Burndown Charts**: Progress visualization
- **Definition of Done**: Completion criteria

## Kanban Method

### Core Principles
- **Visualize workflow**: Make work visible
- **Limit WIP**: Work in progress constraints
- **Manage flow**: Optimize work flow
- **Make policies explicit**: Clear process rules
- **Continuous improvement**: Evolutionary change

### Kanban Board
- **Columns**: Workflow stages (To Do, In Progress, Done)
- **Cards**: Work items or user stories
- **WIP limits**: Maximum items per column
- **Swimlanes**: Categorize different work types
- **Metrics**: Lead time, cycle time, throughput

## Lean Principles

### Lean Thinking
- **Value**: Define customer value
- **Value stream**: Map value-creating activities
- **Flow**: Ensure smooth work flow
- **Pull**: Customer-driven production
- **Perfection**: Continuous improvement

### Waste Elimination
- **Overproduction**: Building unnecessary features
- **Waiting**: Idle time and delays
- **Transportation**: Unnecessary handoffs
- **Over-processing**: Excessive work
- **Inventory**: Unfinished work
- **Motion**: Inefficient movement
- **Defects**: Bugs and rework

## Extreme Programming (XP)

### Practices
- **Pair Programming**: Two developers, one workstation
- **Test-Driven Development**: Write tests first
- **Continuous Integration**: Frequent code integration
- **Refactoring**: Improve code structure
- **Simple Design**: Minimal viable solution

### Values
- **Communication**: Open and frequent communication
- **Simplicity**: Do the simplest thing that works
- **Feedback**: Quick and continuous feedback
- **Courage**: Make necessary changes
- **Respect**: Team member respect

## Agile Estimation

### Techniques
- **Story Points**: Relative sizing
- **Planning Poker**: Consensus-based estimation
- **T-shirt Sizing**: S, M, L, XL categories
- **Affinity Mapping**: Group similar items
- **Three-point Estimation**: Optimistic, pessimistic, most likely

### Velocity
- **Team Velocity**: Story points completed per sprint
- **Capacity Planning**: Future sprint planning
- **Burndown Charts**: Progress tracking
- **Burnup Charts**: Scope and progress visualization
- **Forecasting**: Delivery predictions

## Agile Metrics

### Team Metrics
- **Velocity**: Work completed per iteration
- **Lead Time**: Idea to delivery time
- **Cycle Time**: Development to delivery time
- **Throughput**: Items completed per time period
- **Quality Metrics**: Defect rates, test coverage

### Business Metrics
- **Customer Satisfaction**: User feedback scores
- **Time to Market**: Feature delivery speed
- **ROI**: Return on investment
- **Business Value**: Value delivered to customers
- **Market Responsiveness**: Adaptation to change

## Scaling Agile

### Frameworks
- **SAFe**: Scaled Agile Framework
- **LeSS**: Large-Scale Scrum
- **Nexus**: Scaled Scrum framework
- **Spotify Model**: Squad, tribe, chapter, guild
- **Disciplined Agile**: Process decision framework

### Challenges
- **Coordination**: Multiple team alignment
- **Dependencies**: Cross-team dependencies
- **Communication**: Information flow
- **Culture**: Organizational change
- **Governance**: Compliance and oversight
